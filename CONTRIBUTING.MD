# Contributing to SignalOpen 🚀

First off, thank you for considering contributing to **SignalOpen**! It is people like you who help build **Digital Public Goods** that make information more accessible for everyone.

By contributing, you help us fulfill our mission of providing a free, private, and open-source bridge between critical data and the people who need it.

---

## 🛠️ How Can I Contribute?

### 1. Reporting Bugs
If you find a bug (especially security vulnerabilities related to the Gmail API or Telegram routing), please open an **Issue**:
* Use a clear and descriptive title.
* Describe the exact steps to reproduce the problem.
* Explain which behavior you expected and what actually happened.

### 2. Suggesting Features
We are always looking for ways to improve **SignalOpen**! If you have ideas for new integrations (e.g., adding WhatsApp or Discord support), feel free to open a Feature Request issue.

### 3. Code Contributions
We welcome pull requests for both the **FastAPI** backend and the **Angular** frontend.
* **Backend:** Focus on performance, security, and new "Listener" modules.
* **Frontend:** Help us improve the **Angular Signals** implementation for even better performance on low-end devices.

---

## 🚀 Getting Started

1. **Fork** the repository on GitHub.
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/your-username/SignalOpen.git
   ```
3. **Install Dependencies**:
   * For the backend: `pip install -r requirements.txt`
   * For the frontend: `npm install`
4. **Create a Branch**:
   ```bash
   git checkout -b feature/amazing-new-feature
   ```
5. **Commit your changes**:
   ```bash
   git commit -m 'Add some amazing feature'
   ```
6. **Push to the branch**:
   ```bash
   git push origin feature/amazing-new-feature
   ```
7. **Open a Pull Request**.
